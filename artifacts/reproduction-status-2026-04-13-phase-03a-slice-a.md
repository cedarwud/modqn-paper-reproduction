## What landed

1. **Bundle schema version**: `phase-03a-replay-bundle-v1`, declared in
   `src/modqn_paper_reproduction/export/replay_bundle.py` as
   `BUNDLE_SCHEMA_VERSION` and embedded in every exported manifest and
   provenance map.
2. **Frozen root layout** for the producer bundle:
   ```text
   bundle-root/
   ├── manifest.json
   ├── config-resolved.json
   ├── assumptions.json
   ├── provenance-map.json
   ├── training/
   │   ├── episode_metrics.csv
   │   └── loss_curves.csv
   ├── evaluation/
   │   ├── summary.json
   │   └── sweeps/
   ├── figures/
   │   ├── training-scalar-reward.png
   │   └── training-objectives.png
   └── timeline/
       └── step-trace.jsonl
   ```
3. **Manifest fields** required by Phase 03A §7.2 are produced by
   `export_replay_bundle()`: `paperId`, `runId`, `bundleSchemaVersion`,
   `producerVersion`, `exportedAt`, `sourceArtifactDir`, `checkpointRule`,
   `replayTruthMode`, `timelineFormatVersion`, `coordinateFrame`,
   `replaySummary`, plus `beamCatalogOrder`.
4. **`config-resolved.json`** is now embedded in the bundle by
   `export_replay_bundle()`. The exporter prefers
   `run_metadata.resolved_config_snapshot` (added by `train_main` in this
   slice) and falls back to re-loading `config_path` only when the snapshot
   is absent.
5. **`assumptions.json`** is the resolved-assumption block copied verbatim
   from `run_metadata.resolved_assumptions`.
6. **`provenance-map.json`** is built by `build_provenance_map()` and
   carries a five-category legend
   (`paper-backed`, `recovered-from-paper`, `reproduction-assumption`,
   `platform-visualization-only`, `artifact-derived`) plus per-field
   classifications for paper baseline parameters, recovered figure ranges,
   resolved assumptions, replay timeline truth surfaces, and the
   `manifest.replayTruthMode` selector.
7. **Replay-complete `timeline/step-trace.jsonl`** is generated by a
   deterministic greedy replay over the resolved environment using the
   secondary best-eval checkpoint when available (else the final-episode
   checkpoint). Each row carries every field listed in Phase 03A §7.3:
   `slotIndex`, `timeSec`, `userId`, `userPosition`, `previousServing`,
   `selectedServing`, `handoverEvent`, `visibilityMask`,
   `actionValidityMask`, `beamLoads`, `rewardVector`, `scalarReward`,
   `satelliteStates`, `beamStates`, `kpiOverlay`. Stable identity rules
   are honored: `sat-{idx}`, `sat-{idx}-beam-{local}`, `user-{idx}`, and
   the global `beamCatalogOrder = "satellite-major-beam-minor"`.
8. **Exporter validation** (`validate_replay_bundle()`) is now called at
   the end of `export_training_run()` and rejects bundles that miss any
   required path or any required manifest / timeline-row field. The new
   tests `test_validate_replay_bundle_rejects_missing_manifest` and
   `test_validate_replay_bundle_rejects_missing_timeline_field` cover the
   reject paths.
9. **Sample bundle on disk**: a small reproducible fixture is checked in
   under `tests/fixtures/sample-bundle-v1/`. The fixture is generated by
   `scripts/generate_sample_bundle.py`, which runs the pinned
   one-episode smoke training surface, exports the run through
   `export_main`, then trims the timeline via
   `trim_replay_bundle_for_sample()` to the first user across the first
   ten slots and normalizes machine-specific paths in the manifest. The
   trimmed manifest preserves the source bundle's full row / slot /
   handover counts under `replaySummary.sampleSubset` so consumers can
   distinguish the trimmed sample from a full export.
10. **Tests**: a new `tests/test_replay_bundle.py` covers schema /
    manifest / timeline / provenance / assumption checks against the
    checked-in fixture, a round-trip `train → export → trim → validate`
    test, and the following negative validation tests:
    - missing manifest (`test_validate_replay_bundle_rejects_missing_manifest`)
    - first-row missing timeline fields (`test_validate_replay_bundle_rejects_missing_timeline_field`)
    - empty timeline (`test_validate_replay_bundle_rejects_empty_timeline`)
    - **second** row missing a required field (`test_validate_replay_bundle_rejects_second_row_missing_field`) — this exercises the whole-timeline validation branch, not just the first row
    - malformed JSONL row (`test_validate_replay_bundle_rejects_malformed_json_row`)
    - empty `paperId` (`test_validate_replay_bundle_rejects_empty_paper_id`)
    - symlink target refusal in the trim helper (`test_trim_replay_bundle_refuses_symlink_target`)

    All `pytest tests/` (currently 190 cases) pass after the slice
    landed.

11. **Reviewer-driven hardening** (post first pass):
    - `validate_replay_bundle()` now reads every timeline row (not just
      the first) and rejects malformed JSON or missing required fields
      on any row. It also rejects empty values for the identity manifest
      fields (`paperId`, `runId`, `bundleSchemaVersion`,
      `producerVersion`, `timelineFormatVersion`, `replayTruthMode`).
    - `manifest.coordinateFrame.userPosition` and `beamCenter` now
      explicitly name the tangent-frame anchor ground point, and the
      manifest carries an explicit
      `coordinateFrame.groundPoint = {latDeg, lonDeg}` surface for
      consumers that need the raw anchor without parsing free text.
    - `manifest.slotIndexSemantics = {firstIndex: 1, note: ...}` makes
      the post-step indexing convention explicit so the consumer does
      not have to infer a `slotIndex` offset.
    - `manifest.replaySummary.replaySeedSource` labels which seed
      priority branch (`checkpoint.evaluation_summary.eval_seeds[0]`,
      `run_metadata.best_eval_summary.eval_seeds[0]`,
      `run_metadata.seeds.evaluation_seed_set[0]`, or
      `resolved_config.seeds.evaluation_seed_set[0]`) produced the
      `timelineSeed` used by the deterministic replay.
    - `manifest.replaySummary.slotIndexOffset = 1` mirrors the slot
      index semantics for consumers that want a single numeric offset.
    - `trim_replay_bundle_for_sample()` refuses to overwrite a target
      path that is a symlink or a non-directory existing path.
    - `generate_sample_bundle.py` additionally normalizes
      `evaluation/summary.json` (checkpoint file paths,
      `training_summary.elapsed_s`, and the mirrored `replay_timeline`
      counts) so the shipped fixture is byte-stable across regenerations.

## What did not change

1. The baseline training pipeline, checkpoint rules, comparator sweeps,
   reward-calibration experiment, or `Table II` / `Fig. 3-6` analyses are
   unchanged.
2. No new training experiments were started for Slice A. The smoke
   training run that the test and sample script use is the same
   one-episode pass already exercised by
   `tests/test_sweeps_and_export.py::test_export_cli_emits_bundle`.
3. The reviewed comparison-baseline bundle defined in
   `docs/baseline-acceptance-checklist.md` is unchanged and still the
   authoritative artifact set for downstream comparison.

## How to refresh the sample bundle

```bash
./.venv/bin/python scripts/generate_sample_bundle.py \
    --output tests/fixtures/sample-bundle-v1 \
    --episodes 1 --max-users 1
```

The script normalizes machine-specific paths and `exportedAt` to stable
fixture placeholders so re-runs land on a byte-comparable manifest.
